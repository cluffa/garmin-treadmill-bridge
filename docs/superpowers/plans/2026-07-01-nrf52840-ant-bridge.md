# nRF52840 ANT+ Bridge Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the ESP32 with an nRF52840 that relays treadmill speed to a Garmin watch over ANT+ (native footpod) and receives structured-workout target paces over BLE, removing the phone bridge.

**Architecture:** nRF5 SDK + S340 SoftDevice firmware on a Seeed XIAO nRF52840. The platform-agnostic protocol code in `components/bridge_core/` is reused unchanged; new nRF5/S340 glue modules replace the NimBLE transport. The watch-side ConnectIQ data field is rewritten as a send-only CIQ BLE client. Speed relay moves to an ANT+ Stride SDM master so the watch's only BLE link is the data field's control channel.

**Tech Stack:** C (nRF5 SDK v17.1.0, S340 SoftDevice v7.x), ANT+ SDM device profile, Monkey C (ConnectIQ), GNU Arm Embedded toolchain, `nrfjprog`/J-Link over SWD, host C unit tests (`test/host/`).

## Global Constraints

- **SoftDevice:** S340 v7.0.1 (concurrent BLE central + peripheral + ANT). Do **not** substitute S140 (no ANT) — it breaks the whole design.
- **nRF5 SDK:** v17.1.0. Reference example paths below are relative to the SDK root.
- **Board:** Seeed XIAO nRF52840. No official SDK board file — a `custom_board.h` defines its pin map. No display/battery/button UI.
- **Reuse boundary:** files in `components/bridge_core/` are shared with the ESP boards and MUST stay platform-agnostic — no nRF5/SoftDevice includes may be added to them. All nRF5 glue lives under `boards/xiao-nrf52840/`.
- **ANT+ network key:** required for ANT+ profiles (free from thisisant.com). Stored in `boards/xiao-nrf52840/ant_network_key.h`, which is **git-ignored** (mirrors the existing Garmin-dev-key pattern). A committed `ant_network_key.h.example` documents the format.
- **Control-service BLE interface contract (shared by firmware Task A6 and data-field Part B):**
  - Service UUID: `A6ED0001-D344-460A-8075-B9E8EC90D71B`
  - Control-write characteristic UUID: `A6ED0002-D344-460A-8075-B9E8EC90D71B`, properties: Write and Write-Without-Response.
  - Payload: a UTF-8 ASCII command line using the **existing `ctrl_dispatch` grammar** (e.g. `speed 10.0`, `stop`). The data field converts target pace (m/s) to km/h (`kmh = mps * 3.6`) and writes `speed <kmh>` formatted to one decimal.
  - Advertised device name: `TMILL-CTRL`.
- **ANT+ SDM identity:** ANT device type `124` (0x7C), transmission type `0x05`, RF channel `57` (2457 MHz), channel period `8134` counts (~4.03 Hz), device number = low 16 bits of the nRF `DEVICEID`.
- **Commit discipline:** every task ends with a commit. Host-testable code is TDD (test first, watch it fail, implement, watch it pass). Firmware/hardware tasks use an explicit RTT/hardware acceptance check as their gate.

**Interfaces:**

- Produces:

Page layouts (ANT+ SDM Device Profile; **cross-check bitfields against the profile PDF from thisisant.com before the hardware gate in A2**):

*Page 1 (0x01) — distance & speed:*
| byte | meaning | encoding |
|---|---|---|
| 0 | page number | `0x01` |
| 1 | time fractional | `0` (unused) |
| 2 | time integer (s) | `elapsed_s % 256` |
| 3 | distance integer (m) | `((uint32)distance_m) % 256` |
| 4 | dist-frac(hi nibble, 1/16 m) \| speed-int(lo nibble, m/s) | `((round((distance_m-floor)*16)&0xF)<<4) | (((uint)speed_mps)&0xF)` |
| 5 | speed fractional (1/256 m/s) | `round((speed_mps-floor)*256)` |
| 6 | stride count | `0` |
| 7 | update latency | `0` |

*Page 2 (0x02) — cadence/status:* byte 0 = `0x02`, bytes 1-2 reserved `0xFF`, byte 3 = cadence integer `0`, byte 4 hi nibble cadence-frac `0` / lo nibble speed-int (same as page1 byte4 lo), byte 5 = speed fractional (same as page1 byte5), byte 6 = `0`, byte 7 status = `0x00`.

#include <stdint.h>
#include "model.h"

/* Encode ANT+ Stride-Based Speed & Distance Monitor (SDM) broadcast pages
 * from a treadmill state. Each page is exactly 8 bytes. Pure/stateless. */
void ant_sdm_encode_page1(const treadmill_state_t *s, uint8_t out[8]);

- [ ] **Step 5: Write the implementation** (`ant_sdm_encode.c`):
```c
#include "ant_sdm_encode.h"
#include <math.h>

static uint8_t speed_int(float mps)  { return (uint8_t)((uint32_t)mps & 0x0F); }
static uint8_t speed_frac(float mps) {
    float f = mps - floorf(mps);
    return (uint8_t)lroundf(f * 256.0f) & 0xFF;
}

void ant_sdm_encode_page1(const treadmill_state_t *s, uint8_t out[8]) {
    uint32_t dist = (uint32_t)s->distance_m;
    uint8_t dist_frac = (uint8_t)lroundf((s->distance_m - floorf(s->distance_m)) * 16.0f) & 0x0F;
    out[0] = 0x01;
    out[1] = 0x00;
    out[2] = (uint8_t)(s->elapsed_s % 256);
    out[3] = (uint8_t)(dist % 256);
    out[4] = (uint8_t)((dist_frac << 4) | speed_int(s->speed_mps));
    out[5] = speed_frac(s->speed_mps);
    out[6] = 0x00;
    out[7] = 0x00;
}

void ant_sdm_encode_page2(const treadmill_state_t *s, uint8_t out[8]) {
    out[0] = 0x02;
    out[1] = 0xFF;
    out[2] = 0xFF;
    out[3] = 0x00;
    out[4] = (uint8_t)(0x00 | speed_int(s->speed_mps));
    out[5] = speed_frac(s->speed_mps);
    out[6] = 0x00;
    out[7] = 0x00;
}
```
